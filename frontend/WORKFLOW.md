# Application Workflow

1. **User lands on DiscoveryFeed (/)**: Sees all events, sorted by match percentage based on their skills.
2. **User searches**: Types in SearchBar, filtering events in real-time by title or required skills.
3. **User goes to Dashboard (/dashboard)**: Sees their profile statistics, skills, and top matched events (≥50%).
4. **Admin goes to AdminPanel (/admin)**: Can create new events and see a list of existing ones. New events immediately appear in the DiscoveryFeed.
